# Story 1.4: Meta OAuth 2.0 Flow & Access Token Management

**Epic:** 1 - Infrastructure & Authentication
**Status:** 📋 Draft
**Complexity:** Alta
**Type:** Feature / Integration

---

## Description

Implementar fluxo OAuth 2.0 com Meta/Facebook para conectar contas de anúncios. Usuários fazem login via Facebook, autorizam acesso à conta de anúncios, e token é armazenado criptografado no Supabase.

## Acceptance Criteria

- [ ] Facebook Login button na página de conexão
- [ ] Fluxo OAuth 2.0 completo (request → callback → token armazenado)
- [ ] Access token criptografado no campo `ad_accounts.access_token` (AES-256)
- [ ] Token refresh automático antes de expirar
- [ ] Suporte a múltiplas contas Meta por usuário
- [ ] Status de conexão visível (conectado/expirado/erro)
- [ ] Erro handling gracioso

## Implementation Notes

Fluxo:
1. User clica "Conectar conta Meta"
2. Redirect para Facebook Login
3. User autoriza acesso
4. Callback recebe code + state
5. Backend troca code por access_token
6. Token criptografado com master key
7. Armazenado em ad_accounts.access_token
8. Setup automático de refresh (antes de expirar)

Token deve incluir:
- access_token
- expires_at
- refresh_token (se disponível)
