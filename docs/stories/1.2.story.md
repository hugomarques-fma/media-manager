## Description

Implementar Row Level Security (RLS) policies no Supabase para garantir isolamento de dados por usuário. Nenhum usuário deve conseguir acessar dados de outro usuário, mesmo através de queries diretas.

## Acceptance Criteria

- [ ] RLS habilitado em todas as 11 tabelas
- [ ] Policy: usuários veem apenas suas próprias contas/campanhas
- [ ] Policy: dados históricos isolados por account_id
- [ ] Testes de RLS: falha quando tenta acessar dados de outro user
- [ ] Documentação de policies em `docs/database/rls-policies.md`
- [ ] Nenhuma quebra em queries legítimas

## Implementation Notes

Policies necessárias:
- **ad_accounts:** user_id = auth.uid()
- **campaigns:** via ad_account_id → user_id
- **ad_sets, ads:** cascata via campaign_id
- **campaign_metrics:** isolamento por campaign_id
- **ai_suggestions:** isolamento por account_id
- **rules:** user_id = auth.uid()
- **action_logs:** user_id = auth.uid()
- **notifications:** user_id = auth.uid()
- **campaign_notes:** user_id = auth.uid() or colaboradores
